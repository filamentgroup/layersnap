LayerSnap Demos

LayerSnap is a script that builds on top of Snap SVG to enable simple declarative SVG builds by configuring the SVG file itself–no need to script anything.

An SVG illustration can be animated by grouping graphics as desired and giving those groups IDs to specify their transition type, duration, and delay. The animations in this page are set to auto-play when they scroll into view (using Porthole, which is explained later in this page.). To replay a graphic on this page, you can tap or click it.

How-To

Include jQuery (or Shoestring), SnapSVG, and the layersnap JS and CSS files in the dist folder of this repo.

SVG markup must be included inline in the HTML, wrapped in a <div class="layersnap"></div>. Transitions are defined via the ID attribute of g element children of the svg element.


<div class="layersnap">
  <svg>
    <g id="fade">...</g>
  </svg>
</div>

To have the animation play when the page loads, give the div wrapper the attribute data-layersnap-init.


<div class="layersnap" data-layersnap-init>
  <svg>
    <g id="fade">...</g>
  </svg>
</div>

Transition Examples

Setting Duration

You can set a custom duration on any g element to change it from the default of 800ms. For example, the following example has a g with an ID of rotate-right duration-5000, to make its animation run for 5 seconds.

Custom Delays

You can set a custom delay on any g element to change it from the default, which is just after the prior g finishes its transition. If you set a custom delay, note that it represents the time from the start of the entire animation, so delays can be used to change the order that your groups animate in. For example, the following example has a g with an ID of rotate-right delay-2000, to make its transition delay for 2 seconds before starting.

Delaying Start Time for when the graphic is visible

If you've included Porthole in your page, you can have an animation start when it scrolls into the viewport. All of the animations in this page use Porthole to do just that.

With porthole included, to have the animation play once when it scrolls into the viewport, give the div wrapper the attribute data-scroll-activate.


	<div class="layersnap" data-scroll-activate>
	  <svg>
	    <g id="fade">...</g>
	  </svg>
	</div>

To make it replay every time it enters the viewport, also add the attribute data-scroll-deactivate.


	<div class="layersnap " data-scroll-activate data-scroll-deactivate>
	  <svg>
	    <g id="fade">...</g>
	  </svg>
	</div>

Interactive Animations

Animations can interact with other DOM content by changing a group’s ID attribute to include "toggle-" followed by an ID of an element on the page that should be toggled when that group is clicked. Once clicked, a class of "layersnap-toggle-hide" will be added to the associated element, and a class of "layersnap-toggle-active" will be added to the SVG group. By default, all toggles will receive the hide class, so if you'd like to activate a group from the start, add "toggleactive" to that group's ID.

Building Animations in Adobe Illustrator

If you're editing an SVG in Adobe Illustrator, you can group graphics in the Layers palette and name their layer as if you are setting the group's ID attribute (with the transition name, duration, and delay. When you save your SVG, these layers will become `g` elements with ID attributes matching the layer names you created.

Tips and Caveats

  • In order to use the layer naming technique above, you'll need at least 2 layers in the pallete. Unnamed layers will go ignored by the Animation script.
  • If more than one layer has the same name, Illustrator will add underscores and other characters to the SVG markup to make its ID unique. It'll still appear the same way in Illustrator however, and the animation engine knows to look for these characters and ignore them.
